## 二、必要API接口清单

> 以下接口为skill运行所必须，用户需自行申请并配置到环境变量

### 2.1 QVeris API（必须）

| 项目 | 内容 |
|------|------|
| **用途** | 指数实时行情、新闻情绪数据查询 |
| **API地址** | `https://qveris.cn/api/v1` |
| **环境变量** | `QVERIS_API_KEY` |
| **获取方式** | 联系 QVeris 服务商（https://qveris.cn）申请 |
| **密钥格式** | `sk-cn-xxxxxxxxxxxxxxxxxxxxxxxx` |
| **权限说明** | 需要 `hangseng_polysource.index.livequote.query.v2` 和 `hangseng_polysource.pubsentiment.news.query.v2` 两个tool的执行权限 |
| **额度限制** | 盘中额度受限，耗尽后自动降级到新浪单一数据源 |

### 2.2 微信推送（必须）

| 项目 | 内容 |
|------|------|
| **用途** | 推送日报和盘中预警到用户微信 |
| **环境变量** | `WEIXIN_TARGET_USER` |
| **格式说明** | 微信用户ID，格式：`o9cq80_xxxxxxxxxx@im.wechat` |
| **获取方式** | 在 OpenClaw 的微信推送功能中查看自己的用户ID |
| **备用方案** | 微信推送失败时自动降级到飞书推送 |

### 2.3 飞书推送（可选，降级用）

| 项目 | 内容 |
|------|------|
| **用途** | 微信推送失败时的降级通知渠道 |
| **环境变量** | `FEISHU_TARGET_USER` |
| **格式说明** | 飞书用户ID，格式：`user:02501203084326315375` |
| **获取方式** | 在飞书「我的ID」应用中查看 |
| **注意** | 如不需要飞书降级，可将 `FEISHU_TARGET_USER` 设为与 `WEIXIN_TARGET_USER` 相同的用户ID |

## 三、隐私抹除方案

### 3.1 环境变量配置

```bash
# 复制配置模板
cp .env.example .env

# 编辑 .env，填入真实值
nano .env

# 设置环境变量（Linux/macOS）
export QVERIS_API_KEY=sk-cn-YOUR_KEY_HERE
export WEIXIN_TARGET_USER=YOUR_WEIXIN_USER_ID
export FEISHU_TARGET_USER=YOUR_FEISHU_USER_ID

# 运行skill时加载
source .env && python3 scripts/market_report_v8.py
```

### 3.2 .env.example 模板说明

`.env.example` 文件已包含所有需要的环境变量，用户只需复制并填入真实值。

### 3.3 禁止行为

- ❌ 禁止将真实API Key硬编码到任何 `.py` 文件
- ❌ 禁止将真实用户ID硬编码到任何 `.py` 文件
- ❌ 禁止将 `.env` 文件提交到版本控制（已在 `.gitignore` 中忽略）
- ❌ 禁止在PRD文档中写入真实API Key或用户ID

## 四、抗噪音措施

### 4.1 数据源降级（抗网络波动）

| 层级 | 数据源 | 降级触发条件 | 降级行为 |
|------|--------|-------------|---------|
| 1st | QVeris API | 额度耗尽/网络超时 | 自动降级到新浪 |
| 2nd | 新浪财经 | 接口失败/超时10秒 | 使用腾讯作为备用 |
| 3rd | 腾讯行情 | 所有源均失败 | 返回"数据获取失败"，存档告警日志 |

### 4.2 防频率限制

| 场景 | 措施 |
|------|------|
| QVeris API调用 | 每次请求后 `time.sleep(0.5)`，防止频率超限 |
| 新浪/腾讯请求 | 全部设置 `timeout=10`，超时自动降级 |
| 日报生成 | 幂等存档（检测到当日记录则跳过，避免重复） |

### 4.3 数据验证

| 验证项 | 方式 |
|--------|------|
| 股票价格 | 必须为正数，负数或零视为无效数据 |
| 涨跌幅 | 必须在 ±20% 以内，超出视为异常 |
| K线数据 | 长度不足20条视为数据不足，跳过评分 |
| 市场状态 | 三个ETF多数原则（防止单一ETF误判） |

## 五、Noise Resistance 抗噪音测试用例

### 5.1 数据源失败测试

| 测试场景 | 预期行为 |
|---------|---------|
| QVeris额度耗尽 | 日报正常生成，标注"降级运行"，使用新浪单一数据源 |
| 新浪行情接口失败 | 使用腾讯行情作为备用，正常输出 |
| 腾讯行情也失败 | 返回"数据获取失败"，告警日志记录，不发送空日报 |
| 所有数据源失败 | 静默退出，告警日志记录，等待下次Cron重试 |

### 5.2 股票状态测试

| 测试场景 | 预期行为 |
|---------|---------|
| 股票停牌 | 标注⚪，自动跳过，不阻塞其他股票 |
| 股票退市 | 标注⚪，从股票池移除，下次不再扫描 |
| 涨跌停 | 特殊标注📍，不影响其他股票的处理 |
